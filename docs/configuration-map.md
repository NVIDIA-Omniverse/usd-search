# Configuration Map

One table from feature to the file that defines it, the env var that overrides
it, and the helm value that sets it in Kubernetes. Quickstart compose sets the
same env vars in [`docker-compose.yml`](../docker-compose.yml) and its overlays.

| Feature | Config file (defaults / schema) | Env override | Helm value |
|---|---|---|---|
| LLM/VLM connection (shared default for all roles) | [`llm_client/client.py`](../packages/llm-client/llm_client/client.py) (`LLMConnectionConfig`) | `USDSEARCH_LLM_API_KEY`, `USDSEARCH_LLM_BASE_URL` | `deepsearch.vision_endpoint.provider.*` (secret + base_url) |
| LLM query parsing (query → filters) on/off + model | [`llm_parse/config.py`](../services/deepsearch_api/deepsearch_api/llm_parse/config.py) | `USDSEARCH_LLM_PARSING_ENABLED`, `USDSEARCH_LLM_PARSING_MODEL` (+ `_MAX_TOKENS`, `_TEMPERATURE`, `_TIMEOUT_SECONDS`, `_MAX_TRIES`, `_CACHE_SIZE`, `_REASONING_EFFORT`) | `ngsearch.microservices.search_rest_api.llm_parsing.{enabled,model}` |
| LLM query parsing own endpoint (optional; else shares the connection above) | [`llm_parse/config.py`](../services/deepsearch_api/deepsearch_api/llm_parse/config.py) | `USDSEARCH_LLM_PARSING_BASE_URL`, `USDSEARCH_LLM_PARSING_API_KEY` | `ngsearch.microservices.search_rest_api.llm_parsing.provider.*` (secret + base_url) |
| Which filters LLM query parsing understands | [`search_fields.yaml`](../services/deepsearch_api/deepsearch_api/llm_parse/search_fields.yaml) | `USDSEARCH_LLM_PARSING_FIELDS_FILEPATH` (point at your own copy); per-field property keys via each field's `property_env` | mount a custom file + set the filepath env |
| USD-property grounding for the parser (corpus catalog) | [`usd_property_catalog.yaml`](usd-property-catalog.md) (generated by the `/usd-property-catalog` skill; empty = grounding off) | `USDSEARCH_LLM_PARSING_PROPERTY_CATALOG_FILEPATH` | `ngsearch.microservices.search_rest_api.llm_parsing.property_catalog` (paste the catalog; chart mounts it + sets the env) |
| LLM query parsing system prompt | [`llm_parsing_system_prompt.txt`](../services/deepsearch_api/deepsearch_api/llm_parse/prompts/llm_parsing_system_prompt.txt) | `USDSEARCH_LLM_PARSING_PROMPT_FILEPATH` | mount + filepath env |
| VLM result validation | [`validation/config.py`](../services/deepsearch_api/deepsearch_api/validation/config.py); output schema [`validation_fields.yaml`](../packages/llm-client/llm_client/validation/validation_fields.yaml) | `USDSEARCH_VISION_VALIDATION_{ENABLED,MODEL,MAX_CONCURRENT,TIMEOUT_SECONDS,MAX_TRIES,DOMAIN_CONTEXT_FILEPATH}` | `ngsearch.microservices.search_rest_api.validation.*` (incl. `domain_context`) |
| VLM metadata generation (crawl-time tagging) | [`metadata_generation.py`](../packages/llm-client/llm_client/metadata/metadata_generation.py); field schema [`metadata_fields.yaml`](../packages/llm-client/llm_client/metadata/metadata_fields.yaml) | `USDSEARCH_VISION_METADATA_MODEL` (+ `_MAX_TOKENS`, `_TEMPERATURE`, `_MAX_TRIES`, `_YAML_FILEPATH`) | `deepsearch.vision_endpoint.metadata_model`, `deepsearch.vision_endpoint.metadata_fields` |
| Hybrid search scoring (RRF, field weights) | [`search_backend/models.py`](../services/deepsearch_api/deepsearch_api/search_backend/models.py) (`ScoringConfig`; also overridable per request) | `HYBRID_SEARCH_SCORING_CONFIG_{RRF_CONFIG,HYBRID_TEXT,VECTOR_FIELDS}` (JSON) | `ngsearch.microservices.search_rest_api.hybrid_search.scoring_config.*` |
| Embeddings (SigLIP2 on Triton) | [`siglip2_triton_client/clip.py`](../packages/siglip2-triton-client/siglip2_triton_client/clip.py) | `TRITON_SERVER_URL`, `CLIP_*` / `SIGLIP2_*` | `global.embedding_deployment.*` |
| Explorer WebUI defaults | [`config.jsx`](../services/explorer/src/config.jsx) (CRA build args, baked at image build) | `REACT_APP_DEFAULT_CUTOFF_THRESHOLD`, `REACT_APP_DEFAULT_FILE_EXTENSION_INCLUDE`, `REACT_APP_IMAGE_MAX_DIMENSION`, `REACT_APP_VALIDATION_{MAX_CONCURRENT,MAX_RETRIES,RETRY_DELAY_MS,TIMEOUT_MS}`, `REACT_APP_API_URL` | n/a — set as build args (`scripts/build-explorer.sh`) |

Notes:
- The helm key `deepsearch.vision_endpoint` is a **backward-compatible values
  key name**; the `vision-endpoint` package itself was replaced by
  [`packages/llm-client`](../packages/llm-client/).
- Precedence everywhere: built-in default → file/YAML → environment variable.
- Per-role details and editing guidance: [models-and-config.md](models-and-config.md).
  Filter catalog details: [search-filters.md](search-filters.md).
  Validation details: [vlm-validation.md](vlm-validation.md).
