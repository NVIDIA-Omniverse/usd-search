      helm get all {{ .Release.Name }}

    - Verify storage backend connection and API readiness
      helm test {{ .Release.Name }}

6. Clean Up:
    - To uninstall/delete the USD Search deployment, run:
      helm uninstall {{ .Release.Name }}

    - Rendering jobs created during deployment are not automatically removed on helm uninstall and may occupy resource on the cluster.
      In order to remove them run:
      kubectl delete jobs -n {{ .Release.Namespace }} -l '{{- include "dictToCommaList" ($renderingJobLabels | toYaml) -}}' --field-selector status.successful=0

    - Secrets and Persistent Volume Claims are not automatically removed on helm uninstall. 
      They are retained in case the instance needs to be re-created. 
      The following commands could be used to manually remove them:

      - Secrets:
        kubectl delete secrets -n {{ .Release.Namespace }} $(kubectl get secrets -n {{ .Release.Namespace }} -l 'app.kubernetes.io/instance={{ .Release.Name }}' -o custom-columns=":metadata.name" | awk '{print}' ORS=' ')

      - PVCs:
        kubectl delete pvc -n {{ .Release.Namespace }} $(kubectl get pvc -n {{ .Release.Namespace }} -l 'app.kubernetes.io/instance={{ .Release.Name }}' -o custom-columns=":metadata.name" | awk '{print}' ORS=' ')
        
        # Neo4j PVC does get the same deployment label automatically assigned, therefore a separate command is need to clean it up
        kubectl delete pvc -n {{ .Release.Namespace }} data-neo4j-0
